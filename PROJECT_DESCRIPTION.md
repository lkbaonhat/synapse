To make an app not just a "storage space" but a true "study assistant," the system needs to be designed based on the scientific principles of human memory.

Here is a breakdown of feature groups, from basic to advanced, that you should consider integrating into your project to optimize the acquisition of new knowledge:

### 1. Scientific Study Features (The Soul of the Project)

This section determines the actual learning effectiveness of the application.

* **Spaced Repetition System (SRS):** Based on the Ebbinghaus "Forgetting Curve." The system automatically calculates the schedule and prompts you to review a question right before you are about to forget it. Questions you struggle with will appear frequently, while those you remember well will have their review intervals extended (e.g., tomorrow, 3 days later, 1 week later, 1 month later).
* **Active Recall:** Design an interface that completely hides the answer. Users must actively think and retrieve the answer from their memory before clicking "Show Answer." They then self-rate the difficulty (e.g., Again / Hard / Good / Easy) so the SRS algorithm can schedule the next review.
* **Specialized Study Modes:**
* *Learn:* Introduces unseen cards/questions in small batches (e.g., 10-20 at a time) to prevent cognitive overload.
* *Review:* Only displays questions that are due for review on that specific day, dictated by the SRS algorithm.
* *Cram / Custom Study:* Allows for rapid, intensive review of an entire deck or a specific tag to prepare for an exam. This session does not alter the underlying SRS scheduling history.



### 2. Management & Storage Features (The Foundation)

* **Diverse Question Formats:** Don't limit the app to just Multiple Choice. Support Flashcards (two-sided cards), Cloze Deletion (fill-in-the-blanks), and Free Response (typing the exact answer).
* **Flexible Categorization System:** Allow grouping questions by Folders (Subjects), Decks, and Tags. Tags make it incredibly easy to filter and cross-review interconnected knowledge areas.
* **Multimedia & Formatting Support:** Allow the insertion of images, audio, and support Markdown or code blocks to accommodate various types of knowledge (from language learning to programming).

### 3. Measurement & Motivation Features

* **Statistics Dashboard:** Visual charts displaying the number of questions studied, retention rates, and forecasting the review workload for the upcoming week.
* **Mastery Level:** Each question or deck should feature a progress bar to display its status (from "Newly introduced" to "Mastered").
* **Streaks & Daily Goals:** A system that tracks consecutive study days and sets a target number of questions to complete daily to encourage consistent review habits.

### 4. Extended Utilities

* **Data Import/Export:** A crucial feature that allows users to bulk-upload questions from Excel/CSV files instead of manual entry, as well as export data for backups.
